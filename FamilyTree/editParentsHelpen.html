Edit Parents Help

The Edit Parents dialog is displayed when you click on the 'Edit' button associated with Parents on the edit individual page. This appears as either Edit Parents if there are already defined parents for this individual, or as Add Parents if there are no parents defined yet. The Edit Parents dialog displays a list of the marriages or relationships in which the identified individual is a child.

For each marriage or relationship the dialog displays:

  • The date, if known, of marriage. This can be used to order the sets of parents chronologically.
  • The name of the father. This is a hyperlink to the web page of that individual.
  • The name of the mother. This is a hyperlink to the web page of that individual.
  • A checkbox indicating which of the sets of parents is preferred, and providing an interface for changing this preference. Clicking on a preference box that does not have a check in it sets the selected parents as preferred, and clears the check mark from the former preferred parents. Note that the preference is recorded in the individual's record, so it is not written to the database until you click on the 'Update Individual button of the invoking edit individual page.
  • A button to display the edit dialog for the specific marriage/relationship. This dialog appears in the same window below the list of marriages.
  • A button to delete the specific marriage/relationship.

Below the list of families there is a Add Parents button to add a new set of parents to the individual.

Marriage Edit Dialog

The dialog for editing details about a specific set of parents is displayed in the same window below the list of sets of parents. It is organized into five categories of information:

  • Information about the father, with a blue background.
  • Information about the mother, with a pink background.
  • Information about events related to the relationship.
  • General information about the relationship.
  • A summary of information about the children of this relationship.

Father

The first row in this section displays the given name and surname of the male partner in the relationship. You can alter this information and it will be applied when you click on the Update Family button. Clicking on the Edit Husband button at the end of this row opens the standard edit individual dialog. This button is disabled if you invoked this dialog from the individual page of the male partner.

The next row displays the surname used by the male partner as a result of this marriage.

The next row contains three buttons that are used to change the male partner in this relationship. These buttons are disabled if you invoked this dialog from the individual page of the current male partner.

  • Clicking on Select Existing Husband opens a dialog that permits selecting any male individual in the database as the male partner.
  • Clicking on Create New Husband opens the standard edit individual dialog to permit you to create a new individual as the male partner. If there is already a male partner associated with this marriage then that individual is detached from this marriage.
  • Clicking on Detach Husband detaches the existing male partner from this marriage but does not delete the individual from the database.

Mother

The first row of this section displays the given name and surname of the female partner in the relationship. You can alter this information and it will be applied when you click on the Update Family button. other information about the female partner, click on the Clicking on the Edit Wife button at the end of this row opens the standard edit individual dialog. This button is disabled if you invoked this dialog from the individual page of the female partner.

The next row displays the surname used by the female partner as a result of this marriage.

The next row contains three buttons that are used to change the female partner in this relationship. These buttons are disabled if you invoked this dialog from the individual page of the current female partner.

  • Clicking on Select Existing Wife opens a dialog that permits selecting any female individual in the database as the female partner.
  • Clicking on Create New Wife opens the standard edit individual dialog to permit you to create a new individual as the female partner. If there is already a female partner associated with this marriage then that individual is detached from this marriage.
  • Clicking on Detach Wife detaches the existing female partner from this marriage but does not delete the individual from the database.

Events

This section contains a summary of events associated with this family. This summary includes only the type of event, the date, and the place of the event. You can modify either date and place on this form. More detailed information about the wedding, including source citations and extended textual description can be edited by clicking on the Details button at the end of this line.

There are a number of events that can be recorded about a marriage, for example the Church of Latter Day Saints sealing, formal engagement, divorce, annulment, and whether or not the couple actually got married. These optional facts are summarized in optional lines following the wedding line. To see more information about a specific fact, or to add source citations, or modify any details about a fact, click on the Details button in the specific line. To remove any fact or event completely click on the Delete button in the specific line.

Following the last event is a line containing an Add Event button. Clicking on this button opens a dialog to define a new event. This line also includes an Order Events by Date button.

General Information

The first line of this section displays the internal record number of the marriage in a read-only field. This is used for debugging.

The next line of the dialog contains a selection list which permits you to specify what the current or ending status of the relationship is.

The next line of the dialog contains a selection list which permits you to specify how the married surnames of the partners are assigned. The default is the traditional western rule that the male partner retains his birth surname while the female partner takes her male partner's surname. This selection controls how the married surname fields higher up the dialog behave.

The next portion of the dialog is a rich-text editor that permits you to enter general information about the relationship. Do not record anything here that can be more precisely described using other features of this dialog. In particular do not record any of the information that can be recorded as a marriage related event. You could, for example, use this space to identify the witnesses to the marriage, and the name of the minister as obtained from the civil marriage registration. Because this is a rich-text editor those names could be inserted as links to the individual pages for those individuals. However if you wish to quote from the marriage notice in a newspaper that quote should be recorded in the source citation text.

Below this text editor is a Details button that opens a dialog to specify, in particular, source citations for the text.

Clicking on Edit Pictures opens a dialog that permits you to associate images with this family.

Children

The last major portion of this dialog contains information on the children of this relationship.

This features a table which displays for each child the name of the child, highlighted in blue for males, in red for females, and green for children for whom the sex is unknown, the date of birth, the date of death, an Edit Child button that can be used to open an edit individual dialog to modify any details about the child, and a Detach Child button to detach (not delete) this child from this family.

You can use the Tab key to advance through the input fields for a child, and to set the focus to either of the buttons. Shift-Tab moves backward through the fields. Pressing the Enter key in any of the input fields moves the focus down to the given name field of the next child. If you do this on the last child, an empty child is added to the table, just as if you had clicked on the Add New Child button. The up and down arrow keys can be used to move within a column.

The last line of this section contains an Add Existing Child button that opens a dialog to choose an existing individual in the database to add as a child of this family, and an Add New Child button that opens an edit individual dialog to create a new individual to be added as a child.

Clicking on Order Children by Birth Date sorts the children by birth date. To make use of this feature ensure that you have entered something for the birth date of each child so that even if you do not know the birth date, the children will sort correctly.

The bottom line of the dialog contains two buttons:

  • Clicking on Update Family writes all of the changes you have made to the database and hides the dialog.
  • Clicking on Close

    closes the dialog without applying any changes to the database.
